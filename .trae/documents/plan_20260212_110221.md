# Reduce Project File Count

I will consolidate the project into the absolute minimum number of files while preserving all functionality.

## Current State
- `app.py` (Main Logic + Embedded HTML/JS/CSS) - **Already consolidated**.
- `requirements.txt` (Dependencies) - **Necessary**.
- `emails.txt` (Test data) - **Optional**.
- `validate_emails.py` (Old CLI script) - **Can be removed**.
- `step1_bulk_upload.py` (Old CLI script) - **Can be removed**.
- `deliverable_emails.txt` (Old output) - **Can be removed**.
- `venv/` (Virtual Environment) - **Necessary for execution**.

## Consolidation Plan
The project is *already* extremely consolidated because `app.py` contains the Backend (Flask), Frontend (HTML), Styles (CSS), and Logic (JS) all in one file.

To "clean up" and leave only the essentials:
1.  **Delete Legacy Scripts**: Remove `validate_emails.py`, `step1_bulk_upload.py`.
2.  **Delete Temporary Data**: Remove `emails.txt`, `deliverable_emails.txt`.
3.  **Preserve Core**: Keep only `app.py` and `requirements.txt`.

## Outcome
You will be left with just **two files**:
1.  `app.py`: The entire application.
2.  `requirements.txt`: The installation list.

This is the most compact form possible for a Python web app without packaging it into a binary.

## Execution
- Delete the unnecessary files.
- Verify `app.py` still runs (it will, as it's self-contained).
