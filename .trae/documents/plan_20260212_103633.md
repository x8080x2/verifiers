# Implement B2B/Business Filter

I will add a "Validate for Business (B2B Only)" checkbox to the UI. When enabled, the backend will pre-filter the email list to remove common free/personal email providers (Gmail, Yahoo, AOL, etc.) *before* sending them to the DeBounce API.

## 1. UI Update (`app.py` - Frontend)
- **Add Checkbox**: Place a checkbox below the input area: `[ ] Validate for Business Only (Remove Gmail, Yahoo, etc.)`.
- **Pass State**: Update `startValidation()` to read this checkbox state and append it to the `FormData` as `filter_free=true`.

## 2. Backend Logic (`app.py` - Backend)
- **Free Provider List**: Define a set of common free domains (e.g., `gmail.com`, `yahoo.com`, `hotmail.com`, `outlook.com`, `aol.com`, `icloud.com`, etc.).
- **Filtering Logic**:
  - In the `validate()` route, check if `filter_free` is present in the request.
  - If true, iterate through the parsed `emails` list.
  - If an email matches a free domain:
    - Skip API call.
    - Yield a "skipped" or "filtered" status update to the frontend so the user knows it was removed (e.g., "Skipped (Personal Email)").
    - Do *not* count it as "Valid" or "Invalid", or maybe count it as a separate "Filtered" metric.
  - If it's a business email, proceed with DeBounce validation as normal.

## 3. Execution
- Stop server.
- Modify `app.py` to include the checkbox and the filtering logic.
- Restart server.
