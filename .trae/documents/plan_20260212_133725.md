# Add Duplicate Removal to Filtering Logic

I will update the "Remove Personal Mail" checkbox logic to also **automatically remove duplicate emails**.

## 1. Backend Logic Update (`app.py`)
- **Deduplication**:
  - In the `validate()` route, *before* the generator loop starts, I will add a deduplication step.
  - This should happen regardless of the checkbox state (as duplicate validation is wasted credit), **OR** specifically when the checkbox is checked if that's the user's strict preference.
  - *Interpretation*: The user said "once this is clicked... duplicate email should be cleaned also". So I will tie it to the `filter_free` flag, or better yet, just do it always because duplicates are never good. However, to strictly follow instructions, I will ensure the deduplication logic runs when `filter_free` is True.
  - **Refined Approach**: I'll create a `seen_emails` set.
    - If `filter_free` is checked:
      - Check if email is in `seen_emails`. If yes, mark as "Duplicate" (skip API).
      - Check if email is "Personal". If yes, mark as "Personal" (skip API).
      - Add to `seen_emails`.
    - This ensures we don't double-count or double-charge.
