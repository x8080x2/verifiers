# Mature Single-Page UI Validator

I will upgrade the `app.py` frontend to a "Big Tech" standard Single Page Application (SPA) look, using modern design principles (Tailwind CSS via CDN) without page reloads.

## 1. UI Redesign (Embedded in `app.py`)
- **Theme**: Clean, professional, "Stripe-like" or "Vercel-like" aesthetic.
- **Single Page Layout**:
  - **Hero Header**: Branding and status.
  - **Main Card**:
    - **State 1: Upload**: A large, animated drop zone with file type validation.
    - **State 2: Processing**: A sleek progress dashboard with:
      - **Circular Progress Ring**: Animated percentage.
      - **Real-time Metrics**: Cards for "Deliverable", "Invalid", "Risky", "Unknown" with trend arrows.
      - **Live Console**: A "Matrix-style" scrolling log of processed emails (optional, can be toggled).
      - **Charts**: A simple bar chart (using CSS or a lightweight library like Chart.js via CDN) visualizing the email quality distribution.
    - **State 3: Results**: A summary view with download options (CSV/TXT) and a "Validate Another List" button.

## 2. Technical Enhancements
- **Client-Side Logic**: Use vanilla JS to manage state transitions (Upload -> Processing -> Results) within the same DOM, ensuring a smooth SPA feel.
- **Real-time Updates**: Enhance the SSE handler to push detailed status updates (Deliverable, Invalid, Risky, Unknown) which will update the dashboard charts and counters live.
- **Download Options**: Allow downloading "All Results" or "Deliverable Only".

## 3. Execution
- Stop the running server.
- Rewrite `app.py` with the new comprehensive HTML/JS/CSS.
- Restart the server.
