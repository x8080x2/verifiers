# Merge Input and Results into Single View

I will modify the frontend in `app.py` to display the input area (text box/upload) and the results dashboard side-by-side or stacked in a single view, rather than switching between them.

## 1. UI Redesign
- **Layout**: I will split the main container into two sections (likely side-by-side on desktop, stacked on mobile).
  - **Left/Top Section (Input)**:
    - Text area for pasting emails.
    - File upload button.
    - "Validate" button.
    - This section will remain active even during validation.
  - **Right/Bottom Section (Results)**:
    - Initially shows a "Ready to Validate" or empty state.
    - Once validation starts, it populates with the live metrics, console log, and chart.
    - Progress bar moves to the top of this section.

## 2. Logic Updates
- **Remove View Switching**: Delete the code that hides `view-upload` and shows `view-dashboard`.
- **State Management**: When "Validate" is clicked:
  - Disable the "Validate" button and input area during processing.
  - Clear previous results in the dashboard section.
  - Start the SSE stream and update the dashboard live.
  - Re-enable inputs when complete.

## 3. Execution
- Stop server.
- Update `app.py` with the new single-view layout.
- Restart server.
